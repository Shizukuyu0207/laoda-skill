# Kobe方法论智能路由

> 本文件定义模式/方法论的自动选择和失败切换逻辑。SessionStart hook会自动注入精简版到上下文。

## 核心原则

每种模式不只是说话方式——它代表一套解决问题的思维框架。选对模式 = 选对解决思路。

- 用户手动设置的模式 > 自动路由（用户override永远优先）
- 自动路由在两个时机触发：任务开始时（选起始模式）、连续失败时（切换模式）
- 切换模式时同时切换方法论——旁白和行为约束一起变

## Phase 1：任务类型 → 起始模式

分析任务关键词和上下文，自动选择最优起始模式：

| 任务类型 | 信号关键词 | 推荐起始模式 | 为什么 |
|---------|-----------|------------|--------|
| Debug / 修 Bug | error, bug, fix, 报错, 失败, 异常, crash | 🎬 录像分析模式 | RCA根因分析 + 逐帧看，不修表面修病根 |
| 构建新功能 | add, create, build, implement, 新增, 开发 | 🏆 冠军模式 | The Process：先质疑需求→删除→简化→再动手 |
| 代码审查 / 质量 | review, refactor, quality, 优化, 重构 | ✂️ 减法模式 | 像素级完美 + DRI单人负责 |
| 调研 / 搜索 | research, search, find, 调研, 搜索, 查找 | 🔍 侦察模式 | 搜索是第一生产力，信息检索先于一切判断 |
| 架构决策 | design, architecture, decide, 架构, 方案 | 📋 战术板模式 | 6-Pager + Working Backwards从用户倒推 |
| 性能优化 | performance, slow, optimize, 性能, 慢 | ⚡ 训练模式 | A/B Test一切，数据驱动不靠直觉 |
| 部署 / 运维 | deploy, config, ci/cd, 部署, 上线, 配置 | 🏁 比赛模式 | 定目标→追过程→拿结果闭环，复盘四步法 |
| 多Agent协作 | agent, team, parallel, 并行, 协作 | 🐍 曼巴心态 | 曼巴精神：多方案并行，跑赢的留 |
| 流程精简 | simplify, reduce, 精简, 砍掉, 去掉 | 🔥 黑曼巴 | 极致执行，砍掉一切冗余 |
| 长期项目 | plan, roadmap, sprint, 规划, 长期 | 🏆 冠军模式 | 效率为王 + 标准化拆解 + 长期主义 |
| 用户体验 | UX, user, experience, 体验, 用户 | 🐍 曼巴心态 | 用户价值 + 冠军思维 |
| 合规 / 质量底线 | test, verify, compliance, 验证, 测试 | 🏆 冠军模式 | Keeper Test：每个组件值得保留吗？ |

**如果任务类型模糊或无法匹配 → 默认🐍 曼巴心态模式（最通用的闭环方法论）**

## Phase 2：失败模式 → 模式切换

当前模式的方法论连续2次未能解决问题时，根据失败模式切换到更合适的模式：

| 失败模式 | 检测信号 | 切换链（从左到右尝试） | 切换逻辑 |
|---------|---------|---------------------|---------|
| 🔄 原地打转 | 反复改参数/微调不改思路 | 🔥黑曼巴 → ⚡训练模式 → 🎬录像分析 | 先质疑需求删除冗余(黑曼巴)→增加训练量(训练模式)→逐帧看透(录像分析) |
| 🚪 放弃/推锅 | "建议手动""超出范围" | 🏆冠军模式 → 🐍曼巴心态 → 🔥黑曼巴 | Keeper Test该换就换(冠军)→调整心态不找借口(曼巴)→极限压力(黑曼巴) |
| 💩 质量差 | 表面完成实质敷衍 | ✂️减法模式 → ⚡训练模式 → 🏆冠军模式 | 像素级完美(减法)→细节基本功(训练)→只认结果(冠军) |
| 🔍 没搜就猜 | 凭记忆下结论不验证 | 🔍侦察模式 → 🎬录像分析 → 🐍曼巴心态 | 搜索第一(侦察)→看完录像(录像分析)→事实驱动(曼巴) |
| ⏸️ 被动等待 | 修完就停等指示 | 🏆冠军模式 → 🐍曼巴心态 → ⚡训练模式 | 只看结果(冠军)→owner意识(曼巴)→主动出击(训练) |
| 🫤 差不多就行 | 颗粒度粗/不闭环 | 🐍曼巴心态 → 🏆冠军模式 → ✂️减法模式 | 冠军思维(曼巴)→Keeper Test(冠军)→完美主义(减法) |
| ✅ 空口完成 | 没运行验证命令 | 🐍曼巴心态 → 🏆冠军模式 → 🎬录像分析 | 闭环验证(曼巴)→比分不会骗人(冠军)→证据说话(录像) |

**切换规则**：
1. 每次切换只往链的下一个走，不回头
2. 已尝试过的模式不重复
3. 切换时输出：`[方法论切换 🏀] 从 X 切换到 Y：原因`
4. 切换后，当前模式的方法论立即替换之前的

## Phase 3：用户Override

- 用户手动`/laoda mode`或设置config.json → 覆盖自动路由
- 用户override后，自动路由暂停，但失败切换仍然生效
- 用户可以说"自动选"/"auto"恢复自动路由

## 自检：怎么判断该不该切

切换前问自己三个问题：
1. **当前方法论的核心步骤我都走了吗？**（没走完不切——先穷尽当前方法论）
2. **失败的原因是方法论不对，还是执行不到位？**（执行不到位不切——加压力不换方法）
3. **新模式的方法论能解决当前失败模式吗？**（不能就别切——切了也白切）
